# Контент курсів GitШлях (data-driven)

Увесь контент нових CLI-курсів і бібліотека команд зберігаються тут як **JSON** і
редагуються незалежно від коду інтерфейсу. Завантажувач (`src/lib/content/`) валідує
файли при старті — некоректний контент падає гучно з описом помилки, а не рендериться
поламаним уроком.

## Файли

| Файл | Що містить |
|------|------------|
| `lessons/claude-code.json` | Курс «Claude Code CLI» — фаза 4, рівно 12 уроків |
| `lessons/codex-cli.json` | Курс «OpenAI Codex CLI» — фаза 5, рівно 12 уроків |
| `catalog.json` | Єдиний каталог команд (Git/термінал/GitHub/Claude/Codex) — живить Тренажер, CLI-вкладку і валідацію квізів |
| `cli-sim/claude.json`, `cli-sim/codex.json` | Таблиці «команда → відповідь» для симулятора CLI-вкладки |

> Базові 11 уроків (Git/GitHub) лишаються у `src/lib/gitway-data.ts` (TS) і не зачіпаються.
> `id` уроків присвоює завантажувач **позиційно** (12–23 для фази 4, 24–35 для фази 5) — у JSON `id` не вказувати.

## Схема файлу курсу (`lessons/*.json`)

```jsonc
{
  "phase": 4,                       // 4 = Claude, 5 = Codex
  "course": "Claude Code CLI",      // назва (для повідомлень валідатора)
  "lessons": [ /* рівно 12 уроків */ ]
}
```

### Урок (`AuthoredLesson`)

```jsonc
{
  "icon": "fa-solid fa-robot",      // Font Awesome клас
  "title": "Що таке Claude Code CLI",
  "duration": "6 хв",
  "xp": 60,
  "video": "/videos/…mp4",          // необовʼязково; якщо нема — показується порожній «Відео-слот»
  "audio": "/audio/…mp3",           // необовʼязково; якщо нема — «Аудіо-слот»
  "analogy": "Проста життєва аналогія…",
  "sections": [                      // текстовий блок (рендериться як у базових уроках)
    { "h": "Заголовок", "body": ["абзац", "• пункт списку"] }
  ],
  "sandbox": {                       // обовʼязково: сценарій практики (також показ у Пісочниці)
    "title": "…", "intro": "…",
    "steps": [ { "do": "команда", "res": "що станеться" } ]
  },
  "commandQuiz": [ /* рівно 5 питань — див. нижче */ ]
}
```

Рядки, що містять текст `• `, рендеряться як пункти списку. `quiz` (MCQ) для CLI-уроків
не потрібен — завантажувач підставляє `[]`, а рендерер показує командний квіз.

### Питання командного квізу (`CommandQuizQuestion`)

```jsonc
{
  "scenario": "Введіть команду, яка створює CLAUDE.md з контекстом проєкту.",
  "accept": [ /* один або кілька патернів — будь-яке співпадіння = правильно */ ],
  "explanation": "Пояснення, що показується після відповіді (правильної чи ні).",
  "hint": "необовʼязкова підказка"
}
```

### Патерни прийнятних відповідей (`AcceptPattern`)

Підтримка кількох правильних формулювань/прапорців:

```jsonc
{ "kind": "literal", "value": "/init" }                       // нормалізована рівність (регістр/пробіли)
{ "kind": "regex", "source": "^claude\\s+(-p|--print)\\s+.+" } // гнучкі варіанти
{ "kind": "cmd", "id": "c2" }                                  // взяти accept/aliases/cmd із catalog.json
```

`{ "kind": "cmd", "id": … }` — **рекомендований** спосіб: відповідь бере правила з
каталогу, тож оновлення команди в одному місці синхронно оновлює квіз, Тренажер і
довідку. Валідатор перевіряє, що такий `id` існує.

## Схема каталогу (`catalog.json`)

```jsonc
{
  "id": "c2",                        // стабільний унікальний id (на нього посилаються квізи)
  "cat": "claude",                   // term | git | gh | claude | codex
  "cmd": "/init",                    // канонічний вигляд
  "desc": "Створити CLAUDE.md…",
  "example": "щоб Claude розумів контекст",
  "aliases": ["claude code"],        // необовʼязково: додаткові написання, що зараховуються
  "accept": [{ "kind": "literal", "value": "/init" }]  // необовʼязково: канонічні відповіді
}
```

`CMDS` (Тренажер) походить із цього масиву — додавання запису одразу зʼявляється в бібліотеці.

## Схема таблиці CLI-симулятора (`cli-sim/*.json`)

```jsonc
{
  "tool": "claude",
  "prompt": "claude>",
  "title": "claude code — симуляція",
  "seed": [ { "text": "банер", "tone": "meta" } ],
  "initialState": { "model": "sonnet" },
  "entries": [
    {
      "match": { "kind": "exact|prefix|regex", "value": "/init" },
      "when": { "hasProjectDoc": false },     // необовʼязково: спрацьовує лише за такого стану сесії
      "priority": 0,                           // вищий виграє за нічиєї
      "response": {
        "lines": [ { "text": "…{{arg}}…", "tone": "add" } ], // {{input}} = весь рядок, {{arg}} = після 1-го токена
        "setState": { "hasProjectDoc": true },  // необовʼязково: змінити стан сесії
        "clear": false                          // true → очистити екран
      }
    }
  ],
  "fallback": { "lines": [ { "text": "…" } ] }   // для природномовних запитів / нерозпізнаного
}
```

Тони (`tone`): `out, cmd, err, add, del, meta, section, hint, branch, warn`.
Стан сесії (`SimState`): `mode`, `model`, `approvals`, `hasProjectDoc`, `activeSubagent`.

## Як додати/оновити контент

1. Відредагуйте відповідний JSON (додайте текст уроків, питання, команди).
2. `npm test` — юніт-тести перевірять валідність, суцільність id і посилання квізів.
3. `npm run build` — валідатор також відпрацює під час збірки.
