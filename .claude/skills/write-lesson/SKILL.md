# Урок GitШлях: авторинг контенту

Уроки CLI-курсів — це **data-driven JSON**, який валідується при старті й у тестах. Некоректний контент падає гучно з описом помилки.

## Крок 0 — прочитай схему

`src/content/README.md` — джерело істини. Прочитай його та **сусідні уроки того ж курсу**, щоб зловити тон, рівень і формат.

## Файли

| Файл | Курс | Фаза | id |
|------|------|------|-----|
| `src/content/lessons/claude-code.json` | Claude Code CLI | 4 | 12–23 |
| `src/content/lessons/codex-cli.json` | OpenAI Codex CLI | 5 | 24–35 |
| `src/content/catalog.json` | Каталог команд (посилання квізів) | — | — |

> Базові 11 Git-уроків — у `src/lib/gitway-data.ts` (TS, MCQ). Цей скіл їх **не** чіпає.

## Обовʼязкові інваріанти (їх перевіряє валідатор)

1. **`id` та `phase` в уроці НЕ вказувати** — присвоює завантажувач позиційно.
2. **Рівно 12 уроків** у файлі курсу.
3. **Рівно 5 питань** у кожному `commandQuiz`.
4. Поля уроку: `icon`, `title`, `duration`, `xp`, `analogy`, `sections` (непорожній), `sandbox.steps` (непорожній). `video`/`audio` — опційно.
5. Питання квізу — режим **введення** (`accept`: непорожній масив патернів) **або** режим **вибору** (`options` ≥2 + `correct`). Обидва мають `scenario` і `explanation`.
6. Для `accept` — надавай перевагу `{ "kind": "cmd", "id": "…" }`; посилання на неіснуючий id у каталозі = помилка валідатора. Новий id спершу додай у `catalog.json`.

## Шаблон уроку

```jsonc
{
  "icon": "fa-solid fa-robot",
  "title": "…",
  "duration": "6 хв",
  "xp": 60,
  "analogy": "Проста життєва аналогія…",
  "sections": [
    { "h": "Заголовок", "body": ["абзац", "• пункт списку"] }
  ],
  "sandbox": {
    "title": "…", "intro": "…",
    "steps": [ { "do": "команда", "res": "що станеться" } ]
  },
  "commandQuiz": [
    { "scenario": "…", "accept": [ { "kind": "cmd", "id": "c2" } ], "explanation": "…", "hint": "…" },
    { "scenario": "…", "options": ["/init", "/clear"], "correct": 0, "explanation": "…" }
    // … рівно 5
  ]
}
```

## Перевірка (обовʼязково після правок)

```bash
npm test              # кількість уроків, суцільність id, посилання квізів, валідність
npm run build         # валідатор також відпрацьовує при збірці (опційно)
```

Якщо тест червоний — повідомлення вказує курс / урок / питання. Виправ причину, не глуши помилку.

## Порада

Для обʼємного авторингу кількох уроків або генерації квізів делегуй агенту **lesson-author** (він тримає тон і схему). Далі проженеш `npm test` тут.
